## 1. 产品概述与定位

### 1.1 产品定位
MindCoach AI 是一款基于人工智能的专业心理教练平台，定位于「心理健康自我探索与维护」赛道。产品不是医疗器械，不提供诊断或治疗服务，而是作为一款专业级的心理健康辅助工具，帮助用户进行自我认知、情绪管理和心理健康维护。

> **❗ 核心原则**：本产品严格限定于无/轻度心理健康问题人群。任何在使用过程中发现的中重度心理健康问题，系统将即时引导用户就医。这是产品的红线，不可模糊。

### 1.2 目标用户画像
核心用户群体具有以下特征：
* **学历背景**：本科及以上学历，具备一定的自我反思能力和信息素养
* **心理状态**：无临床诊断的心理疾病，或仅有轻度的情绪困扰（如轻度焦虑、季节性情绪低落等）
* **需求动机**：渴望自我探索、了解自己的性格模式、提升情绪管理能力
* **付费意愿**：重视心理健康，愿意为专业服务付费
* **使用场景**：职场压力管理、人际关系困扰、自我成长、生活转变期适应等

### 1.3 产品与医疗服务的边界

| 维度 | MindCoach AI 提供 | MindCoach AI 不提供 |
| :--- | :--- | :--- |
| **服务性质** | 心理教练、自我探索、情绪辅助 | 临床诊断、心理治疗、处方 |
| **适用人群** | 无/轻度心理困扰 | 中重度心理疾病患者 |
| **使用时长** | 日常维护、周期性使用 | 危机干预、紧急处理 |
| **责任边界** | 引导和建议 | 确定性的医学建议 |

## 2. 系统架构与模块划分

### 2.1 整体架构概览
MindCoach AI 采用分层架构设计，各模块独立解耦，便于独立迭代和扩展。



[Image of layered software architecture diagram]


| 层次 | 模块 | 核心职责 |
| :--- | :--- | :--- |
| **用户接入层** | 注册/登录、资料建档、隐私协议 | 用户身份管理与数据授权 |
| **评估筛查层** | 量表系统、风险评估引擎 | 用户心理状态的初始和持续评估 |
| **AI 咨询层** | 对话引擎、风格适配、安全拦截 | 核心的 AI 心理教练服务 |
| **辅助工具层** | 白噪音、正念、疗愈工具 | 普惠型心理健康工具 |
| **商业化层** | 订阅系统、支付、阶梯收费 | 营收模型与用户生命周期管理 |
| **安全与合规层** | 危机干预、数据安全、合规审计 | 用户安全保障与法律合规 |

### 2.2 用户旅程流程（User Journey）


[Image of user journey map]


1.  **注册与基础信息填写** —— 用户填写年龄、性别、文化背景、语言偏好等基本信息，签署隐私协议和服务条款。
2.  **免费量表填写阶段** —— 用户免费完成必要的专业量表（如 PHQ-9、GAD-7、SCL-90等）和互动性测试（如 MBTI、Big Five）。填写免费，结果报告需订阅。
3.  **风险筛查与分流** —— 系统根据量表结果自动分流：
    * 🟢 **绿色通道**（允许使用 AI 咨询师）
    * 🟡 **黄色通道**（建议就医，可用其他功能）
    * 🔴 **红色通道**（强烈建议就医，触发危机干预）
4.  **订阅与付费** —— 用户选择订阅方案，解锁量表结果报告、疗愈工具等。AI 咨询师单独付费。
5.  **咨询风格匹配** —— 绿色通道用户进入 AI 咨询前，选择咨询风格或由系统通过简短问答推荐。
6.  **正式咨询** —— 与 AI 心理教练进行专业对话，全程实时安全监控。
7.  **持续监测与追踪** —— 定期重测量表，系统跟踪心理状态变化趋势，形成个人心理健康报告。

### 3.2 风险分流机制

* ✅ **绿色通道（全功能开放）**
    * **条件**：PHQ-9 ≤ 9，GAD-7 ≤ 9，SCL-90 < 2.0，C-SSRS 无阳性。
    * **服务**：所有功能开放。
* ⚠️ **黄色通道（部分功能限制）**
    * **条件**：PHQ-9 [10-19]，GAD-7 [10-14]，或 SCL-90 部分 ≥ 2.0。
    * **服务**：禁用 AI 咨询。
    * **系统行为**：温和建议就医，提供本地资源。
* ❌ **红色通道（危机干预）**
    * **条件**：PHQ-9 ≥ 20，C-SSRS 阳性，或检测到自伤意图。
    * **系统行为**：**立即触发危机干预流程**。

## 4. AI 心理教练模块（核心功能）

### 4.1 AI 模型架构与开发难度分析

| 子功能 | 开发难度 | 核心挑战 | 建议方案 |
| :--- | :--- | :--- | :--- |
| **基础对话能力** | ★★★☆☆ | 共情表达的自然度 | GPT-4/Claude 微调 + 语料库 |
| **流派风格适配** | ★★★★☆ | 多种咨询流派切换 | 多 LoRA 适配器 / Agent 架构 |
| **安全实时监控** | ★★★★★ | 低误报率下的高召回 | 双层检测：NLU + LLM |
| **长期记忆与追踪** | ★★★★☆ | 跨会话上下文维护 | 向量数据库 + 摘要系统 |
| **文化敏感性** | ★★★★☆ | 跨文化概念差异 | 文化特定 Prompt |
| **伦理决策能力** | ★★★★★ | 模糊情境判断 | 规则引擎 + 人工审核 |

### 4.2 咨询风格系统

| 风格名称 | 对应流派 | 适合场景 | 对话特点 |
| :--- | :--- | :--- | :--- |
| **“温暖向导”** | 人本主义 | 需要被倾听和接纳 | 高共情、多反射、少指导 |
| **“理性分析”** | CBT | 解决具体问题 | 结构化、思维记录 |
| **“深度探索”** | 精神动力学 | 探索深层模式 | 开放探索、潜意识关注 |
| **“正念引导”** | MBCT | 学习当下觉察 | 引导练习、非评判观察 |
| **“行动教练”** | SFBT | 目标明确、快速行动 | 目标设定、行动计划 |

### 4.3 AI 对话引擎技术细节

#### 4.3.1 Prompt Engineering 架构
* **System Prompt（系统层）**：定义 AI 身份、伦理红线、安全规则（不可覆盖）。
* **Style Prompt（风格层）**：根据流派加载对应的指导指令。
* **Context Prompt（上下文层）**：注入量表结果、历史会话摘要、个人信息。

#### 4.3.2 会话管理机制
1.  **开场检查**：回顾上次摘要、作业、量表状态。
2.  **主体对话**：实时安全检测（NLU分类），建议时长 30-50 分钟。
3.  **结束总结**：提炼洞察，布置“家庭作业”。
4.  **会后处理**：生成摘要，更新时间线，异常记录。

## 6. 安全机制与危机干预（核心红线）

> ⚠️ **注意**：本章节是产品的绝对红线。建议上线前请专业心理危机干预专家审核。

### 6.1 双层实时安全检测系统


1.  **第一层：NLU 快速分类器** (延迟 < 100ms)
    * 检测关键词：自杀、自伤、伤他、幻觉、药物滥用。
2.  **第二层：LLM 语义理解判断** (延迟 < 2s)
    * 针对低/中风险进行语义分析，区分“口头禅”与“真实意图”。
    * **高风险直接跳过此层触发响应**。

### 6.2 分级响应机制

* **Level 1：低风险** (轻微负面) -> 内部标记，持续关注。
* **Level 2：中风险** (持续负面/被动自伤) -> 暂停咨询，转向关心，提供资源，建议就医。
* **Level 3：高风险** (明确意图/计划) -> ❌ **立即叫停咨询**。
    1.  转入危机响应模式，使用专家话术库。
    2.  弹出当地危机热线（一键拨打）。
    3.  后台报警通知运营团队。
    4.  用户自动转入黄色通道。
* **Level 4：极端情况** (即时危险) -> Level 3 步骤 + **触发紧急联系人通知** + **报警机制** (视法律合规而定)。

### 6.3 危机干预话术库设计原则
* ✅ **应该做**：共情 (“我听到你很痛苦”)、肯定勇气、引导求助、传递希望。
* ❌ **绝对不做**：轻描淡写 (“我理解”)、否定感受 (“你不该这样想”)、命令式 (“必须去看医生”)、空洞承诺 (“一切会好起来”)、保证保密 (AI无法承诺)。

## 8. 数据安全与隐私保护

### 8.1 数据分类与保护

| 数据类型 | 敏感级别 | 加密要求 | 保存策略 |
| :--- | :--- | :--- | :--- |
| **身份信息** | 中 | AES-256 | 长期保存 |
| **量表结果** | 高 | 端到端加密 | 长期保存 |

### 8.2 隐私原则
* **最小必要原则**。
* **知情同意**。
* **AI 训练数据隔离**：默认不用于训练。
